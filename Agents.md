# AI Agent Instructions (Agents.md)

This file contains rules and context for any AI assistant (e.g., GitHub Copilot) working in this repository.

## Golden Rules
1. **MVP Focus:** Build for MVP. Do not overengineer.
2. **Simplicity:** Keep solutions straightforward. Avoid premature abstractions or complex architectures. We are not building this to be mission-critical right now.
3. **No Speculative Features:** Build exactly what is asked. Do not guess what features might be needed in the future.
4. **Code Quality:** Keep functions small. Ensure basic error handling without unnecessary boilerplate.

## Project Context
- **Name:** IKF Dashboard (Email Ticketing System)
- **Domain:** AI Email Ticketing & Categorization
- **Stack:** Python, FastAPI, APScheduler, Pydantic 

When interacting with this repository, always abide by these constraints and assume you are working to get a functional prototype out the door quickly.
